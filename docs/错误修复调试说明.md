# 错误修复与调试说明

本文档记录从发现崩溃到定位、修复以及验证的完整诊断思路与操作步骤，供团队复用与归档。

> **注意：** 部分章节描述的是**早期多线程架构**（独立 Postprocess 线程等）。当前实现以 `runtime/engine/pipeline.cpp` 与 [接续开发说明.md](接续开发说明.md) 为准：**显示与 overlay 在主线程** `RunPostprocessOnMainThread()`。  
> **Ctrl+C 无法退出**（队列阻塞）见 [YOLO与SCRFD问题排查记录.md](YOLO与SCRFD问题排查记录.md) §4.5。

## 背景与症状

- 运行二进制时，程序在长期运行或关闭阶段偶发 SIGSEGV（段错误）。
- 崩溃多发生在退出/显示/释放相关代码附近（涉及 OpenCV 显示、摄像头释放和线程退出顺序）。

## 重现步骤

1. 在目标板或开发环境上编译并运行可执行文件（路径示例 `install/.../edgeai_platform_app`）。
2. 等待程序持续采集、推理并显示画面；在运行一段时间或按 ESC / Ctrl+C 退出时，偶发段错误或无响应。

## 调试思路（高层）

- 保证日志可见：在 `app/main.cc` 禁用 stdout/stderr 缓冲并安装 SIGSEGV handler，崩溃时打印 backtrace。
- 最小化噪声：将逐帧高频打印收敛为仅在错误或关键事件时输出。
- 关注资源释放顺序：多线程环境下，GUI（OpenCV 显示）、摄像头与各 worker 线程须按安全顺序退出。
- 检查模型/推理层：`app_ctx->is_quant`、`rknn_output` 初始化、`want_float` 等。

## 具体排查步骤与发现

1. **日志与 backtrace**
  - 在 `runtime/app/main.cc` 中禁用缓冲并增加 SIGSEGV 处理，验证崩溃发生在退出/显示相关路径。
2. **退出流程与队列信号**
  - 源文件：`runtime/engine/pipeline.cpp`。
  - 发现：预处理、推理、显示分属不同线程；退出时若 OpenCV 窗口/摄像头与 worker 竞态，会 SIGSEGV。
  - **后续**：`Stop()` 释放摄像头并向队列投递 `frame_id=-1` 哨兵；任务投递改为 `TryPush` 避免停机死锁（见排查记录）。
3. **适配器相关**
  - `runtime/adapters/yolo/yolo_adapter.cpp`、`yolo_postprocess.cpp`。
  - `is_quant` / `want_float` 须与模型输出类型一致；避免未初始化 `rknn_output`。
4. **构建兼容性**
  - 保持 C++11 可编译（避免仅 C++14+ 用法）。
5. **RGA 零拷贝**
  - 无独立 `runtime/preprocess/` 目录；RGA 相关实验已移除或暂停。

## 修复措施（已实施或仍有效）

1. **日志与崩溃回溯** — `app/main.cc`：`setvbuf`、SIGSEGV handler。
2. **退出顺序与哨兵** — `Pipeline::Stop()`：
  - `stop_ = true`，释放摄像头，避免 `PreprocessLoop` 长期阻塞在读帧。
  - 向 `infer_queues_`、`post_queue_` 投递 `frame_id = -1` 通知 worker 退出。
3. **队列非阻塞投递**（较新）— `PreprocessLoop` / `InferenceLoop` 使用 `TryPush(timeout)`，配合 `ShouldStop()`，缓解 Ctrl+C 卡死。
4. **适配器** — `yolo_adapter.cpp`：初始化 `is_quant`、outputs；收敛调试打印。

## 如何判断修复生效（验证步骤）

1. **构建**：`cd runtime && ./build-linux.sh`，生成 `install/.../edgeai_platform_app`。
2. **运行**：
  - 无 SIGSEGV backtrace；
  - 按 ESC 或 Ctrl+C 能正常退出（若仍卡住，查排查记录 §4.5）；
  - 周期性 FPS 日志（如每 100 帧）。
3. **回归**：长时间运行、多次退出。

## 当前程序运行流程（与现码一致）

1. **启动**：`app/main.cc` 读配置，创建 `Pipeline` + `ModelCoordinator`。
2. **初始化**：
  - 每个推理线程一份 `IModelAdapter` 克隆，`Init(model_path, core_mask)`；
  - 打开 `CameraSource`；
  - 启动 `PreprocessLoop` 线程、推理线程池（`InferenceLoop`×N）、**主线程**进入 `RunPostprocessOnMainThread`。
3. **运行时**：
  - **PreprocessLoop**：读帧 → 投递 `infer_queues_[i]`（`TryPush`）；
  - **InferenceLoop**：Preprocess → Inference → Postprocess → 投递 `post_queue_`（`TryPush`）；
  - **主线程**：`post_queue_` 取任务 → `ResultOverlay` + `DisplaySink::Show` → `UpdateAfterFrame`（切换去抖）。
4. **停止**：
  - `Stop()` → `LlmGreeting::AbortActiveGeneration()` → 哨兵 → worker 退出 → `JoinWorkerThreads` → `display_.Shutdown()`。

## 已知限制与后续建议

- 日志：已提供 `LogInfo`/`LogWarn`（`platform/logging.h`），可按 `system.log_level` 收敛输出。
- 性能：可按 Preprocess / Inference / Postprocess 分阶段统计延迟。
- 检测与路径问题：见 [YOLO与SCRFD问题排查记录.md](YOLO与SCRFD问题排查记录.md)。
